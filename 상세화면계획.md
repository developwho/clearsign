
## ClearSign 화면 상세 설계

### 전체 구조: 4개 화면 (+ 보너스 1개)

```
S0 인트로 → S1 업로드 → S2 로딩 → S3 분석 뷰 (메인)
                                        ├─ 문서 뷰 (어노테이션)
                                        ├─ 위험 요약 헤더
                                        ├─ 조항 디테일 패널
                                        └─ 행동 스크립트 패널
```

---

### S0. 관점 전환 인트로

**목적:** 데모 첫 10~15초. 심사위원의 공감을 건다.

**레이아웃:**
- 풀스크린 다크 배경
- 중앙에 계약서 텍스트 블록 — 핵심 조항이 일본어 법률용어 + 뒤섞인 어순으로 표시
- 하단 타이포: "글자는 보입니다. 의미는 도달하지 않습니다."
- CTA 버튼 or 자동 전환 → S1으로 이동

**구현 난이도:** 낮음 (정적 HTML/CSS)

---

### S1. 업로드

**목적:** 빠른 진입. 3초 안에 파일을 넣을 수 있어야 함.

**레이아웃:**
- 중앙 정렬, 수직 스택
- 상단: ClearSign 로고 + 한 줄 태그라인
- 중앙: 드롭존 (점선 보더, 드래그앤드롭 + 클릭 업로드)
  - 아이콘 + "계약서를 올려주세요" + 지원 포맷(PDF, JPG, PNG)
- 하단: **"샘플 계약서로 체험하기"** 텍스트 버튼 — 데모 시 이걸로 원클릭 진입
- 전체적으로 여백 넉넉하게, AI Studio 랜딩 느낌

**핵심 인터랙션:**
- 파일 드롭 시 즉시 S2로 전환 (업로드 확인 단계 없음)

---

### S2. 분석 로딩

**목적:** AI가 실제로 일하고 있다는 신호 + 기대감 빌드

**레이아웃:**
- 중앙에 계약서 썸네일/미리보기
- 그 위로 스캔라인 애니메이션이 위→아래로 지나감
- 하단에 진행 단계 텍스트 (순차 등장):
  ```
  "계약서를 읽고 있습니다..."
  "조항을 분석하고 있습니다..."
  "표준 계약서와 비교하고 있습니다..."
  "위험 구간을 표시하고 있습니다..."
  ```
- 각 단계가 체크 표시로 전환되며 진행

**목표 소요:** 실제 API 응답 시간(3~10초)을 자연스럽게 소화

---

### S3. 분석 뷰 (메인 — 가장 중요)

이 화면이 ClearSign의 전부입니다. 업로드 후 사용자가 머무는 유일한 작업 화면.

**전체 레이아웃: 3-Zone 구성**

```
┌─────────────────────────────────────────────────────────┐
│  [위험 요약 바]  ₩12,400,000 위험  │ 위험 3건 │ 주의 2건 │
├──────────────────────────┬──────────────────────────────┤
│                          │                              │
│     문서 뷰 (좌측)        │      디테일 패널 (우측)       │
│     ~60% 너비             │      ~40% 너비              │
│                          │                              │
│  계약서 텍스트 전문        │  [기본] 위험 조항 리스트      │
│  위험 구간 하이라이트       │  [클릭 시] 조항 상세 설명    │
│  - 빨강: 고위험            │  [행동] 협상 메시지/경고      │
│  - 주황: 주의              │                              │
│  - 초록: 안전              │                              │
│                          │                              │
├──────────────────────────┴──────────────────────────────┤
│  (하단 액션 바 — optional)                                │
└─────────────────────────────────────────────────────────┘
```

**Zone 1: 위험 요약 바 (상단 고정)**
- 좌측: **"잃을 수 있는 최대 금액: ₩12,400,000"** — 페이지 최상단, 항상 보임
- 우측: 위험/주의/안전 조항 카운트 뱃지 (빨강 3, 주황 2, 초록 5)
- 전체 이탈도 게이지 또는 등급(A~F)도 가능
- 높이: 48~56px, 슬림하게

**Zone 2: 문서 뷰 (좌측 60%)**
- 계약서 텍스트를 **재구성된 텍스트 뷰**로 렌더링 (PDF 원본 아닌 AI 파싱 결과)
  - 이유: 하이라이트/인터랙션 자유도, 반응형 텍스트 크기, 로딩 속도
- 위험 조항 구간에 **인라인 하이라이트** (배경색)
  - 빨강 계열: 이탈도 70%+ (고위험)
  - 주황 계열: 이탈도 40~70% (주의)
  - 하이라이트 안 된 구간: 안전
- 하이라이트 구간 우측에 **이탈도 뱃지** (예: `87%`)
- 하이라이트 클릭 → 우측 패널이 해당 조항 디테일로 전환
- 현재 선택된 조항은 하이라이트가 더 진하게 + 좌측 보더라인 강조

**Zone 3: 디테일 패널 (우측 40%)**

3가지 상태를 전환:

**상태 A — 기본 (개요)**
- 위험 조항 리스트 (이탈도 높은 순 정렬)
- 각 항목: 조항명 + 이탈도 % + 1줄 요약 + 위험 색상 인디케이터
- 항목 클릭 → 문서 뷰에서 해당 위치로 스크롤 + 상태 B로 전환

**상태 B — 조항 상세 (쉬운 한국어)**
- 상단: 조항명 + 이탈도 뱃지
- **3단계 탭:**
  - Level 1: 쉬운 말
  - Level 2: 비유 + 아이콘
  - Level 3: 구체 시나리오 (기본 선택)
- 변환된 설명 텍스트
- 원문 비교 토글 ("원문 보기" — 접을 수 있는 영역)
- 하단 CTA: **"이 조항에 대해 행동하기"** → 상태 C로 전환

**상태 C — 행동 스크립트**
- 이탈도 ≤60%: 수정 요청 메시지 (편집 가능 텍스트 + 복사 버튼)
- 이탈도 >60%: 경고 배너 + 전세사기 체크리스트 + 신고처 연결
- **"목록으로 돌아가기"** → 상태 A

### Stitch 작업 단위 (디자인 생성 순서)

| 순서 | 대상 | Stitch에 요청할 핵심 | 복잡도 |
|------|------|---------------------|--------|
| 1 | S3 전체 레이아웃 | 3-zone 레이아웃 + 문서 하이라이트 + 우측 패널 | 높음 |
| 2 | S3 패널 상태 A | 위험 조항 리스트 | 중 |
| 3 | S3 패널 상태 B | 3단계 탭 + 쉬운 한국어 | 중 |
| 4 | S3 패널 상태 C | 행동 스크립트 (협상/경고 분기) | 중 |
| 5 | S1 업로드 | 드롭존 + 샘플 버튼 | 낮음 |
| 6 | S2 로딩 | 스캔 애니메이션 | 낮음 |
| 7 | S0 인트로 | 풀스크린 타이포 | 낮음 |
